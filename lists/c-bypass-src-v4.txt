# c-bypass-src-v4.txt
# LAN IPv4 addresses that should NEVER be proxied (source-based bypass).
# Matching clients get nft `return` — all their traffic goes direct.
#
# Typical candidates:
#   - IoT devices (smart plugs, sensors) that don't need a proxy
#   - game consoles (latency matters; proxy adds RTT)
#   - guest devices running their own corporate VPN
#   - a control laptop where you want A/B comparison vs proxied traffic
#
# For per-device *forcing to a specific outbound* (not bypass), use xray
# routing instead — add a rule in xray/50-routing.json.tpl:
#
#   { "type": "field", "inboundTag": ["c-def-in"],
#     "source": ["192.168.1.50/32"], "outboundTag": "T" }
#
# That rule, placed above the domain-based rules, forces the device's
# traffic through outbound T regardless of destination.
#
# Format: one LAN IP per line. # starts a comment. Empty lines ignored.
#
# Examples (uncomment to enable):
# 192.168.1.50           # PlayStation 5
# 192.168.1.60           # Samsung TV (IoT, keep direct)
